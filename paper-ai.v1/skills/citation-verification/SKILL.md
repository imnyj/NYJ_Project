# Citation Verification Skill

## When to load
- Librarian after collecting refs.json entries
- Reviewer (QA mode) auditing `drafts/main.tex` citations
- Before any paper submission

## Why this matters
Literature reports 14–95 % BibTeX invalidity across LLM outputs
(GhostCite, Aug 2025). A rejected submission over a fake citation is common.
This skill provides **mechanical, deterministic** checks — no LLM judgment.

## Never-do list

1. ❌ Never let an LLM generate BibTeX strings directly.
2. ❌ Never cite a paper the agent "remembers" — always re-verify via DOI.
3. ❌ Never trust author lists generated by the model; fetch from Crossref.
4. ❌ Never skip retraction check.

## ID-only citation protocol

Enforced downstream. The agent emits only:
```
\cite{corpusID:270214}
\cite{doi:10.1109/TVT.2023.1234567}
```
Not full BibTeX. The post-processor fetches authoritative metadata.

## Verification pipeline

For every citation in `refs.json`, run:

### Step 1: DOI resolve
```python
import requests

def verify_doi(doi: str, mailto: str) -> dict | None:
    """Returns metadata or None if DOI doesn't resolve."""
    r = requests.get(
        f"https://api.crossref.org/works/{doi}",
        headers={"User-Agent": f"paper-ai/0.1 (mailto:{mailto})"},
        timeout=10,
    )
    if r.status_code != 200:
        return None
    return r.json()["message"]
```

If the DOI does not resolve: **the citation is a ghost**. Drop it.

### Step 2: Metadata cross-check
Compare `refs.json` entry against Crossref's authoritative response:
- title (fuzzy match, Levenshtein < 20 % diff)
- first author surname (exact)
- year (exact)
- container-title (venue, normalized)

Mismatches → Librarian must re-verify by fetching the paper again.

### Step 3: Retraction check
```python
def check_retraction(doi: str) -> bool:
    """Retraction Watch database via Crossref."""
    r = requests.get(
        f"https://api.labs.crossref.org/works/{doi}",
        timeout=10,
    )
    if r.status_code != 200:
        return False  # inconclusive
    data = r.json().get("message", {})
    return data.get("update-type") == "retraction" or \
           "retraction" in str(data).lower()
```

Retracted papers → issue a high-confidence QA report.

### Step 4: Publisher whitelist
Check that the `container-title` or URL domain is on the Librarian SCIE
whitelist (IEEE / Elsevier / ACM / Nature / MDPI / Springer / Wiley / Taylor
& Francis / APS / Oxford / Cambridge). Reject everything else **including
arXiv / bioRxiv / ResearchGate** — these are policy-excluded sources.

### Step 5: Claim-support check (sentence-BERT)

For every `\cite{...}` in the draft, extract the surrounding sentence and
measure its semantic overlap with the cited paper's abstract:

```python
from sentence_transformers import SentenceTransformer, util

model = SentenceTransformer("allenai/specter2_base")  # scientific sBERT

def claim_supports(citation_sentence: str, abstract: str,
                   threshold: float = 0.35) -> bool:
    embs = model.encode([citation_sentence, abstract], convert_to_tensor=True)
    sim = float(util.cos_sim(embs[0], embs[1]))
    return sim >= threshold, sim
```

Threshold 0.35 is a practical default for SPECTER2 — tune per-corpus.
Low similarity (< 0.2) → citation is likely misused ("right paper, wrong
claim" error common in LLMs).

## QA report format (Reviewer, QA mode)

```markdown
## Citation audit

### High-severity (confidence >= 0.9)
- [L142] `\cite{corpusID:123456}` — DOI does not resolve. Drop or re-verify.
- [L203] `\cite{doi:10.XXXX/...}` — Retraction flag. Remove.

### Medium-severity (0.7 <= confidence < 0.9)
- [L88] `\cite{corpusID:789012}` — Year mismatch (stated 2022, Crossref says 2021).
- [L310] `\cite{corpusID:345678}` — Claim-abstract similarity = 0.18 (< 0.35).
                                     Sentence claims X, cited paper about Y.

### Low-severity
- [L57] Reference order inconsistent with IEEEtran style (first use).
```

## Integration call chain

```
Writer emits .tex with \cite{corpusID:N}
        ↓
Phase-5 post-processor: fetch metadata per ID → build refs.bib
        ↓
Reviewer (QA) runs verify_doi / check_retraction / claim_supports
        ↓
High-severity issues → block submission, loop back to Writer/Librarian
```

This fully deterministic pipeline means citation fidelity does NOT depend on
any LLM's memory, and is verifiable by re-running the script.
