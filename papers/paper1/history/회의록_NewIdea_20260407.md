## 1. 회의 배경

기존 ST-CVAE 모델을 단순 MLP와 비교 실험한 결과 성능 차이가 미미했다. 이를 반영하여 CVAE를 폐기하고 결정론적 Multi-Branch Attention 구조(ST-MBAN)를 새로 설계했으며, 해당 설계의 타당성·효율성·효과성·적합성을 3개 에이전트가 독립 검토했다.

---

## 2. 에이전트별 리뷰 요약

### 2-1. Critic (기술 타당성)

**지지하는 설계 결정:**
- 3-Branch 입력 분리 (K/T/S) — 이질적 변수군 간 간섭 방지, 구조적 근거 명확
- Cyclical Encoding (T Branch) — 신호등 주기 불연속 문제 직접 해결
- 출력 3개→2개 단순화 — 파생 가능 변수 제거, 학습 집중도 향상
- Huber Loss — 이상치(돌발 정체) 강건성 확보

**수정 권고 (⚠️):**
1. **Social Branch Feature Attention 명칭**: 입력이 고정 tabular 변수라면 Self-Attention이 아닌 "Gated Feature Weighting"에 가깝다. 심사자 반론 가능성 있음. 구조적 근거를 수식으로 명시하거나 명칭을 정확히 정의해야 함.
2. **Fusion Multi-Head Attention Q/K/V 미정의**: Z_k/Z_t/Z_s를 단순 Concat 후 Self-Attention에 넣으면 Branch 구분 없이 처리됨 → **3개 토큰(Branch별 1 토큰)으로 분리한 Token-level Attention** 구조를 권고.
3. **ResBlock 필요성 ablation 부재**: 출력이 스칼라 2개인 단순 회귀에서 ResBlock 2개가 필요한지 검증 필요. 심사자 반드시 지적할 항목.
4. **SUMO 신규 변수 실환경 대응 논거 부재**: Simulation-derived vs. Sensor-derived 분류 명시 필요.

**재검토 필요 (❌):**
- **ST-CVAE ≈ MLP 원인 미분석 상태에서 구조 전환**: 실패 원인이 데이터 부족이면 구조 복잡도 증가가 역효과를 낼 수 있음. 원인 규명 선행 필요.
- **ReGLU 선택 근거 부재**: Vehicular prediction 도메인에서 ReLU/GELU 대비 우위 미검증. 선택 이유 명시 또는 비교 실험 필요.

### 2-2. Researcher (논문 기여도)

| 기여 | 논문 기여도 | 핵심 강점 | 보완 필요 사항 |
|------|------------|----------|---------------|
| ① CVAE→결정론적 전환 | 중간 | 실험 기반 동기 부여, 이론 낭비 제거 | 체류시간 단봉분포 + 물리적 결정론성 이론 논거 추가 필요 |
| ② 3-Branch 분리 | 중간~높음 | CCVN 특화 의미론적 분류, 해석 가능성 | "왜 3분류가 최적인가" ablation 필요 |
| ③ Multi-Head Attention 융합 | 중간 | 지배 인자 자동 선택, 해석 가능 가중치 | Branch별 attention weight 시각화로 수치 입증 필요 |
| ④ SUMO 신규 변수 | 낮음~중간 | 미시 교통지표 체계적 도입 | SHAP + leave-one-out ablation 필수 |
| ⑤ 출력 단순화 | 낮음 | Precaching에 직접 매핑 | 단독 기여로는 약함. 설계 결정으로 내부 처리 권고 |

**핵심 포지셔닝 전략:**  
"새로운 기법 발명"이 아닌 **"CCVN 체류시간이 물리적으로 결정론적이라는 도메인 특성 위에서, K/T/S 세 인자의 gradient conflict를 구조적으로 해소한 특화 설계"** 로 프레이밍. 기여 5는 논문 기여 목록에서 제외하고 System Model 절의 설계 결정 사항으로 처리.

### 2-3. Simulator (구현 가능성)

**핵심 결론:**
- 구현 전반 중간 난이도. 특별한 기술적 블로커 없음.
- **가장 먼저 결정해야 할 사항:** Fusion Attention 입력 형태
  - Option A: Z_concat 단일 벡터 → Self-Attention (구현 단순, 효과 불명확)
  - Option B: [Z_k, Z_t, Z_s] 3-토큰 시퀀스 → Token-level Attention (구조적 근거 명확, 차원 통일 필요)
  - **권고: Option B** (Critic과 동일 견해)

**ST-CVAE ≈ MLP 원인 가설 (3가지):**
1. KL Collapse — KL annealing 미적용 시 latent Z*가 노이즈 수렴
2. 신호 주기성 미처리 — Cyclical Encoding 없이 이산값 처리 → 비선형 패턴 포착 실패
3. 타겟 중복 — t_dwell/t_ent_n/t_lev_n의 강한 선형 관계 → 복잡 모델이 오히려 모순 예측

**SUMO 신규 변수 9개 수집 가능성:** 모두 가능.
- 직접 API: `dist_leader`, `v_leader`, `q_len`, `occ`, `est_travel_time`
- 우회 구현 필요: `n_ahead`, `v_ahead_avg`, `n_merge_nxt`, `route_lane_changes`

**최소 필수 실험 (4개):**
1. ST-MBAN vs. MLP (기존 변수 동일) — 구조 효과 단독 검증
2. ST-MBAN (기존) vs. ST-MBAN (신규 변수 포함) — 변수 기여도
3. ST-MBAN vs. ST-CVAE — CVAE 폐기 정당화
4. Branch 제거 Ablation (K/T/S 각 조합) — Branch 분리 기여도

| 항목 | 내용 |
|------|------|
| **예측 트리거** | 차량이 RSU에 Content를 요청하는 순간에만 Snapshot 예측 실행 (이벤트 기반, 주기적 실행 아님) |
| **학습 구조** | 각 RSU가 자신의 통신범위 내 데이터만으로 독자적 학습 |
| **목적 1** | 교차로별 지역 특성 반영 (신호 주기, 도로 구조, 합류 패턴이 RSU마다 다름) |
| **목적 2** | 중앙 서버 불필요 → 추론 지연시간 최소화 |

이 컨텍스트는 다음 설계 결정을 **추가 정당화**한다:
- **3-Branch 구조**: RSU마다 K/T/S 변수의 중요도가 다르므로, 분리 인코딩이 지역 특성을 더 잘 포착
- **Feature-wise Attention (S Branch)**: RSU별 독립 학습 → 같은 변수라도 RSU마다 다른 attention weight → 지역 교통 패턴이 가중치에 반영됨
- **결정론적 예측**: 이벤트 기반 단발성 추론에서 CVAE의 stochastic sampling은 불필요한 지연

## 6. 3개 결정 사항 최종 확정

### 결정 (1): Fusion Attention 구조 → **3-토큰 Token-level Attention 확정**
- Z_k / Z_t / Z_s 각각 1개 토큰으로 취급
- 입력 형태: `(batch, 3, d_branch)`
- 각 Branch 출력 차원 `d_branch` 통일 필요
- 근거: Branch 간 상호작용을 구조적으로 구분, 논문 논거 명확

---

### 결정 (2): Social Branch Feature Attention → **Feature-wise Attention 유지, 수식 명확화 적용**

**Critic 판단:** RSU 독립 학습 컨텍스트에서 명칭 및 구조 모두 기술적으로 타당. 명칭을 "Feature-wise Attention"으로 정확히 하고 수식 명시.

**Researcher 판단:** SENet(Squeeze-and-Excitation Network, Hu et al., 2018)과 동일 구조 → 선행 연구 인용으로 심사자 반론 차단. 단순 MLP 교체 시 3-Branch 설계 서사가 약해지므로 유지 권고.

**확정된 수식 정의:**

$$\mathbf{a} = \sigma(W_2 \cdot \text{ReLU}(W_1 \cdot \mathbf{x}^S + b_1) + b_2), \quad \mathbf{a} \in [0,1]^{d_S}$$
$$Z_s = \mathbf{a} \odot \mathbf{x}^S$$

- $\mathbf{a}$: RSU별 독립 학습되는 feature-wise attention weight
- $\odot$: element-wise product
- SENet의 SE Block과 동일한 논리 구조 → 인용 가능

---

### 결정 (3): 출력 단순화 (3→2) → **설계 결정 확정 + System Model에서 정당화**

**실험 근거 (사용자 제공):** 기존 ST-CVAE에서 t_ent_n과 t_lev_n을 분리 예측했을 때 두 값의 불확실성이 중첩(uncertainty compounding)되어 성능이 저하됨.

**정당화 논거 (Researcher 작성):**
> "The prior work predicts three separate time points — current dwell time, next RSU entry time, and next RSU departure time — as independent outputs, which introduces **uncertainty compounding**: the prediction errors of entry and departure times accumulate when computing the usable sojourn duration. We address this by directly targeting the two semantically self-contained dwell durations, $\tau_\text{cur}$ and $\tau_\text{nxt}$, which are the quantities of direct relevance to precaching decisions. This reformulation eliminates error accumulation across dependent predictions and removes physically inconsistent outputs that arise from independently regressing correlated time points."

**배치 결정:**
- Introduction 기여 목록에서 **제외**
- System Model의 예측 타겟 정의 문단에 **통합** (위 영어 서술 삽입)
- 기여 ①(CVAE 폐기) 동기 서술의 2차 근거로 **흡수**
