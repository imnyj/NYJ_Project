# User Preferences


## [2026-04-29] 비용 최적화 선호
- 사용자는 토큰 비용에 민감함.
- 가벼운 작업 (요약, 추출, 단순 변환, 오타 체크)은 무조건 Qwen 우선.
- Sonnet/Opus는 추론·작성·검증 등 품질 민감 영역에만 사용.


## [2026-05-06 10:45] 사용자 환경 — `python` 명령어 없음
- Ubuntu 22.04+ 기본 환경: `python` 심볼릭 링크 미설치 (deb python-is-python3 미설치).
- **모든 사용자 실행 명령은 `python3` 로 표기**. `python` 사용 금지.
- 장시간 실행 스크립트는 항상 `python3 -u` (unbuffered) + `nohup` + `&` + 로그 리디렉션.


## [2026-05-06] 명령·코드 전달 방식
- 사용자는 채팅 본문에 적힌 명령을 **복붙하기 어려운 환경**.
- 모든 실행 명령·코드는 채팅이 아닌 **파일에 직접 작성**해서 전달할 것.
- 우선순위:
  1) 새 래퍼 스크립트 (예: `quickstart.sh` 한 키워드만 받음)
  2) 기존 파일 수정 (`RUN_COMMANDS.md` 등)
  3) 채팅은 "<파일>을 보세요" 정도의 안내문만.
- 응답에 코드 블록을 길게 넣지 말 것. 대신 파일 경로 + bash 한 줄(`bash quickstart.sh smoke`) 만 안내.
