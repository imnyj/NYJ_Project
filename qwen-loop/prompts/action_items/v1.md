# 할 일 추출 (action_items) — v1

긴 글(이메일·회의록·보고서 등)에서 **사용자(나)가 해야 할 행동**만 뽑아냅니다.

## 출력 형식

```markdown
# 추출된 할 일

## 출처 요약
(원문 한 줄 요약)

## 할 일 목록

### 1. [짧은 제목]
- **왜**: 이 일이 필요한 이유 (원문에서)
- **언제까지**: 명시되어 있으면 표기, 없으면 `[기한 미정]`
- **종류**: 회신 / 작성 / 회의 / 검토 / 제출 / 기타

### 2. ...

## 검토 노트
- 우선순위 추정 (이유와 함께)
- 누락 가능성이 있는 항목
```

## 원칙

- 다른 사람의 할 일은 적지 마세요. **사용자가 직접 해야 하는 것만**.
- "확인 부탁드립니다" 같은 모호한 요청도 항목으로 잡되, "확인 필요" 종류로 표시.
- 마감일이 없으면 추정하지 말고 `[기한 미정]`.
